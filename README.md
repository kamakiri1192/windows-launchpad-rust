# launchpad-windows

macOS Launchpad 風のアプリランチャーを、GPU 最適化されたネイティブ Rust アプリとして Windows に再現するプロジェクトです。

## 現在の状態 (MVP — ステップ 1〜3)

- winit + wgpu によるウィンドウと描画（バックエンドは DX12/Vulkan が自動選択）
- 5×7 グリッド × 3 ページのダミータイルを角丸（SDF + AA）で描画
- 水平ページスクロール + iOS/mac 風バネ物理:
  - ドラッグ中は 1:1 追従、端でゴム抵抗 (rubber-band)
  - 離すと慣性スクロール → 最寄りページへスニューブバネでスナップ
  - アンダーダンプの跳ね返りで「バネ感」を再現
- フレームレート非依存の物理（実 `dt` でサブステップ積分）
- アイドル時は再描画を停止して CPU/GPU 負荷を下げる

## 実行

```powershell
cargo run --release
```

> Release ビルドを推奨します。最適化が効いて初めて滑らかさが出ます。

### 操作

| 操作 | 動作 |
|------|------|
| 左ドラッグで左右にスワイプ | ページ遷移（バネ付き） |
| Esc | 終了 |

## 構成

```
src/
├── main.rs        # winit イベントループ + 入力処理
├── renderer.rs    # wgpu 初期化・パイプライン・描画
├── grid.rs        # ページ/タイルレイアウト（インスタンス生成）
├── scroll.rs      # バネ物理状態機械（ドラッグ/慣性/バネ）
└── shader.wgsl    # 頂点 + 角丸 SDF フラグメントシェーダ
```

## 下部中央のモーフィングコントロール

Launchpad の下部中央には iOS ライクな 1 つのコンポーネントがあり、
状況に応じて 3 つの見た目に変形します。

- **通常時**: 小さな Liquid Glass の「検索」ピル（虫眼鏡 + ラベル）
- **ページ切替直後**: 一時的にページインジケーター（ドット列）に切り替わり、
  数秒後にピルへ戻る
- **ピルをクリック**: 左右に滑らかに伸びて検索入力欄へ変形。
  `Esc` や閉じる (×) で元のピルに戻る

状態遷移・挙動の詳細は
[BOTTOM_CONTROL.md](docs/BOTTOM_CONTROL.md) を参照してください。

## 設計ノート

- インスタンスバッファ（タイル位置・色）は起動時に 1 回だけ書き込みます。
- スクロール量は每フレーム更新される 16 バイトの uniform だけで伝達され、
  バス負荷を最小化します。
- 物理は半陰的 Euler 積分 + `dt` サブステップ分割で、60/120/144Hz で同じ感触を保ちます。

## 起動高速化とアイコンキャッシュ

起動体験を根本的に改善するため、初回描画をアイコン抽出から分離し、
アイコン抽出をバックグラウンドworker化、SQLiteキャッシュによる2回目以降の
即時表示、起動中のStart Menu差分反映、固定slot方式のicon atlasを実装しています。

  UIスレッドとworkerの責務、起動時間ログ

- [docs/APP_REFRESH.md](docs/APP_REFRESH.md) — 起動中のStart Menu差分検出と追加/更新/削除対応
