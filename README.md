# launchpad-windows

macOS Launchpad 風のアプリランチャーを、GPU 最適化されたネイティブ Rust アプリとして Windows に再現するプロジェクトです。

## 現在の状態 (MVP — ステップ 1〜3)

- winit + wgpu によるウィンドウと描画（バックエンドは DX12/Vulkan が自動選択）
- 5×7 グリッド × 3 ページのダミータイルを角丸（SDF + AA）で描画
- 水平ページスクロール + iOS/mac 風バネ物理:
  - ドラッグ中は 1:1 追従、端でゴム抵抗 (rubber-band)
  - 離すと慣性スクロール → 最寄りページへスニューブバネでスナップ
  - アンダーダンプの跳ね返りで「バネ感」を再現
- フレームレート非依存の物理（実 `dt` でサブステップ積分）
- アイドル時は再描画を停止して CPU/GPU 負荷を下げる

## 実行

```powershell
cargo run --release
```

> Release ビルドを推奨します。最適化が効いて初めて滑らかさが出ます。

### 操作

ランチャーは**常駐型**です。ウィンドウを閉じても（Esc、他ウィンドウのクリック、アプリ起動）
プロセスは生きたままなので、いつでも即座に再召喚できます。本終了はトレイアイコンから。

| 操作 | 動作 |
|------|------|
| `Win+Space` | **どこからでもランチャーを召喚**（ローレベルキーボードフックでこのコンビのIME切り替えを抑制） |
| 左ドラッグで左右にスワイプ | ページ遷移（バネ付き） |
| アプリアイコンをクリック | アプリを起動してランチャーを隠す |
| 他ウィンドウをクリック / Alt-Tab | ランチャーを自動で隠す |
| Esc | ランチャーを隠す（常駐のまま）。検索フィールドが開いていれば最初のEscはフィールドを閉じるだけ |
| トレイアイコンを左クリック / 右クリック→「表示」 | 召喚 |
| トレイアイコンを右クリック→「終了」 | **本終了** |

## 構成

```
src/
├── main.rs              # winit イベントループ + 入力処理
├── platform_windows.rs  # 常駐ライフサイクル（Win+Space フック + トレイアイコン）
├── renderer.rs          # wgpu 初期化・パイプライン・描画
├── grid.rs              # ページ/タイルレイアウト（インスタンス生成）
├── scroll.rs            # バネ物理状態機械（ドラッグ/慣性/バネ）
└── shader.wgsl          # 頂点 + 角丸 SDF フラグメントシェーダ
```

## ランチャーライフサイクル（常駐化）

このアプリは macOS Launchpad / PowerToys Run 風の常駐型ランチャーとして
振る舞います。

- **Win+Space で召喚**: 専用スレッド上の `WH_KEYBOARD_LL` フックがこの
  コンビネーションを検出し、システムのIME切り替えを抑制した上で
  `UserEvent::Summon` を投げてウィンドウを表示します。スタートメニューが
  同時に開くのを防ぐため、コンボ消費後に無害なダミーキー(`VK_F20`)を
  投入して「Win単独タップ」と判定されないようにしています。
- **フォーカス消失で自動非表示**: 他ウィンドウをクリック / Alt-Tab で
  `WindowEvent::Focused(false)` を受け取り `hide()` します。ただし召喚直後
  500ms 以内のフォーカス喪失は `SetForegroundWindow` の過渡期とみなし無視します。
- **常駐**: Esc / アプリ起動 / CloseRequested は `event_loop.exit()` せず
  `hide()` に切り替えています。プロセスは生き続け、アイコンワーカや
  リフレッシュウォッチャも稼働し続けます。
- **トレイアイコンで本終了**: 右クリックメニューの「終了」が唯一の本終了経路で、
  `UserEvent::QuitRequested` → `std::process::exit(0)` で即時終了します
  （OS がプロセス終了時にLLフックとトレイアイコンを解放します）。

## 下部中央のモーフィングコントロール

Launchpad の下部中央には iOS ライクな 1 つのコンポーネントがあり、
状況に応じて 3 つの見た目に変形します。

- **通常時**: 小さな Liquid Glass の「検索」ピル（虫眼鏡 + ラベル）
- **ページ切替直後**: 一時的にページインジケーター（ドット列）に切り替わり、
  数秒後にピルへ戻る
- **ピルをクリック**: 左右に滑らかに伸びて検索入力欄へ変形。
  `Esc` や閉じる (×) で元のピルに戻る

状態遷移・挙動の詳細は
[BOTTOM_CONTROL.md](docs/BOTTOM_CONTROL.md) を参照してください。

## 設計ノート

- インスタンスバッファ（タイル位置・色）は起動時に 1 回だけ書き込みます。
- スクロール量は每フレーム更新される 16 バイトの uniform だけで伝達され、
  バス負荷を最小化します。
- 物理は半陰的 Euler 積分 + `dt` サブステップ分割で、60/120/144Hz で同じ感触を保ちます。

## 起動高速化とアイコンキャッシュ

アイコン抽出をバックグラウンドworker化、SQLiteキャッシュによる2回目以降の
即時表示、起動中のStart Menu差分反映、固定slot方式のicon atlasを実装しています。
